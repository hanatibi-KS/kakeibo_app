# 家計簿サポートアプリ 🐱
https://hanatibi-ks.github.io/kakeibo_app/

**満足度で振り返る、続けられるお小遣い帳。**

「いくら使ったか」だけでなく **「その支出に満足したか」** を記録することで、
ムダ遣いに自分で気づけるようにした家計簿アプリです。
スマホのホーム画面に追加すれば、アプリのように全画面・オフラインで使えます（PWA）。

サーバー・データベース不要。HTML / CSS / JavaScript だけで動きます。

---

## ✨ 特徴（ほかの家計簿アプリとの違い）

### 1. 満足度で振り返る
支出ごとに `◎満足 / ○普通 / △いまいち` を記録します。
明細画面では **満足度△の支出を「ムダ遣い候補」として自動で集計** し、
「合計◯円ぶんが"いまいち"でした」と振り返れます。

金額の大小ではなく **"払った価値があったか"** で家計を見直せるのが狙いです。

### 2. 予算ペース予測 ＋ 応援キャラ
登録画面のトップで、**経過日数から月末の支出を予測**します。
猫のキャラクターがペースに応じて反応します。

| 状態 | キャラ | メッセージ例 |
|---|---|---|
| 余裕あり（予測 80%未満） | 😺 | 余裕があるニャ♪ 残り 12,000 円 |
| ちょうどいい（〜100%） | 🐱 | ちょうどいいペース！ |
| 少し早め（〜120%） | 😿 | 月末は約 3,200 円オーバー予測 |
| 使いすぎ（120%超 / 超過） | 🙀 | このペースだと月末に約 8,000 円オーバーしそう… |

「1日あたり約◯円まで使えるよ（残り◯日）」の目安も表示されます。

## 📱 画面

### 登録画面（`index.html`）
- 今月のペース ダッシュボード（予測・応援キャラ・使用率バー）
- 今月のお小遣い登録
- 支出登録（日付 / 項目 / 金額 / カテゴリ / **満足度**）
- 今月の残高表示

### 明細画面（`list.html`）
- 月別の支出一覧（`←` `→` で月を移動）
- 合計支出・残高
- **ムダ遣い候補**（満足度△の集計）
- カテゴリ別の円グラフ（割合%付き・表示/非表示切替）
- 編集（モーダル）・削除

### 健康管理画面（`health.html`）
- 運動の記録（筋トレ / ストレッチ / ウォーキング / ランニング / その他 ＋ 時間）
- 今月の運動回数・合計時間・連続日数、応援キャラ🐱
- **健康への投資** — 支出カテゴリ「健康」の合計と、
  「運動1回あたりのコスト」を表示（動くほどおトクになる、家計簿ならではの動機づけ）
- 月別の履歴・削除

## 🚀 使い方

### PCで動かす

`index.html` をブラウザで開くだけでも動きます。
ただし **Service Worker（オフライン機能）は `localhost` か HTTPS でしか動かない** ため、
PWAとして正しく確認するにはローカルサーバー経由で開いてください。

```bash
python3 -m http.server 8765
# → http://localhost:8765/index.html
```

サーバーを止めるとき:

```bash
lsof -ti:8765 | xargs kill
```

### スマホのホーム画面に追加する

1. GitHub Pages などの **HTTPS** で公開する
   （リポジトリの `Settings` → `Pages` → Branch を `main` に設定）
2. スマホのブラウザで公開URLを開く
3. **iPhone**: 共有ボタン → 「ホーム画面に追加」
   **Android**: メニュー → 「アプリをインストール」
4. ホーム画面の🐱アイコンから、全画面・オフラインで起動します

> ⚠️ 同じWi-Fi内のPCサーバー（`http://192.168.x.x:8765`）にスマホから接続した場合、
> 画面表示は確認できますが **HTTPSではないためオフライン機能は動きません**。

## 💾 データ構造（LocalStorage）

データベースは使わず、ブラウザの LocalStorage に保存しています。

### `expenses` — 支出の配列

```json
[
  {
    "id": "m3x9k2a7bc",
    "date": "2026-07-17",
    "item": "昼食",
    "amount": 800,
    "category": "食費",
    "month": "2026-07",
    "satisfaction": "ok"
  }
]
```

| キー | 内容 |
|---|---|
| `id` | 一意のID。**編集・削除はこのIDで対象を特定する**（並び順に依存しない） |
| `date` | `YYYY-MM-DD` |
| `item` | 項目名 |
| `amount` | 金額（数値） |
| `category` | `食費` / `交通費` / `娯楽費` / `健康` / `その他` |
| `month` | `YYYY-MM`（月別表示の絞り込み用。`date` から自動生成） |
| `satisfaction` | `good`（◎） / `ok`（○） / `bad`（△） |

> 古いデータに `id` や `month` が無い場合は、`list.js` の `normalizeExpenses()` が自動で補完します。

### `allowances` — 月別のお小遣い

```json
{ "2026-07": 30000, "2026-08": 25000 }
```

**お小遣いは月ごとに独立しています。**
新しい月になったら **必ず0円スタート**で、自動の繰り越しはしません
（毎月あらためて予算を決める、という設計方針）。

### データを消したいとき

ブラウザのDevTools（F12）→ Console で:

```js
localStorage.removeItem("allowances");  // お小遣いだけ消す
localStorage.removeItem("expenses");    // 支出だけ消す
localStorage.clear();                   // 全部消す
```

## 🔧 開発メモ

- **CSSは外部ファイルにせず、各HTML内の `<style>` に記述する**（方針）
- HTMLとJavaScriptは分離する
- UIはシンプル・直感的を優先。カード型・中央配置・緑基調・角丸・影
- レスポンシブ対応済み（固定幅を廃止し `max-width` + `clamp()` で伸縮）

### ファイルを修正したあとの注意（PWA）

Service Worker がファイルを保存しているため、修正しても反映されないことがあります。
`service-worker.js` の **キャッシュ版数を上げてください**。

```js
const CACHE_NAME = "kakeibo-v2";  // → "kakeibo-v3" に変更する
```

## 🗺 今後のロードマップ

### Ver2: レシート読み取り（OCR）
レシートを撮影 → 文字認識 → 金額を抽出 → 確認画面 → 家計簿へ登録

- 候補技術: Tesseract.js（ブラウザ内で動くためサーバー不要）
- ⚠️ 日本語レシートのOCRは難易度が高い（感熱紙の滲み・特殊フォント・店ごとに違うレイアウト）
- 方針: **OCRは"下書き"と割り切り**、必ず確認画面でユーザーが修正できるようにする。
  まずは精度の出やすい **金額（数字）の抽出**から狙う
